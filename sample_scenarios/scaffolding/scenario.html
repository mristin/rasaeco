Scaffolding

Summary

A wrong planification of scaffolding. Wrong height prevents the proper fixing of the reception platform. Workers not authorized to make any changes.

Models

ðŸ”—plan/main

This is the main model of the site plan covering the whole site. It is updated on demand, as the plan changes.

ðŸ”—observed/main

This is the main model representing the digital twin of the building. It is updated daily.

ðŸ”—staff

This is the model capturing the information about the site personnel. It is updated in real time.

Definitions

scaffoldLabel = IfcLabel("Scaffold")

scaffolds =

Scenario

As-planned

The scaffolds are expected to be tracked in the model plan/main. The plan should include the position and the height of the scaffolds.

As-observed

The possible placements for the reception platform should be computed based on the observed/main.

As-planned vs As-observed

During the planning phase, the scaffolds are wrongly planed. planning The receptionPlatforms can not be appropriately fixed on the sitesite. construction

ðŸ”—misplacedScaffolds

Formally, we select the scaffolds with incorrectly planned height:

misplacedScaffolds = 
    SELECT s
    FROM
        rp in receptionPlatforms
        s in scaffolds
    WHERE
        abs(s.NominalHeight - rp.NominalHeight) < 1 meter

Analytics

The misplacedScaffolds should be reported on the web platform.

Scheduling

All the tasks affected by the misplacedScaffolds should be set to blocking.

Formally:

tt = SELECT t 
    FROM
        s in misplacedScaffolds
        t in IfcTask modeled in plan/main
    WHERE
        Affected(t, s)

blockedLabel = IfcLabel("blocked")

UPDATE
    t.Status = blockedLabel
FROM
    t in tt

Safety

No worker is allowed to make changes to the misplacedScaffolds:

FROM
    w in workers
    s in misplacedScaffolds
MUST
    not CanModify(w, s)

We consider only the scaffolding on a single construction site.site
